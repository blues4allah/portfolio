I follow a data-driven process,
guided by collaboration

1. Discovery

Learn about the problem

Conduct customer discovery interviews. Analyze current product use. Collect customer feedback from internal teams. Read Productboard portal comments. Run usability studies. Hold focus groups. Analyze competitors and market status.

2. Roadmapping

Weigh solutions & make a plan

Set organizational goals. Set up key performance indicators. Brainstorm possible solutions. Prioritize features based on how they contribute to goals. Define user personas and goals. Consider alternative solutions to underlying problem. Hold internal discussions with team leaders. Define minimum viable product. Get development estimates and plan timeline.

3. Development

Design & iterate

Design wireframes, mock-ups, and clickable prototypes. Seek internal team and external stakeholder feedack. Communicate prototype to engineers. Design reusable front-end components. Refine CSS. Run A/B tests. Conduct usability studies. QA new features on different browsers and devices.

4. Results

Measure outcomes

Anounce release to customers and stakeholders. Observe analytics and track key performance indicators. Set up trainings for users. Collect user feedback. Conduct usability studies. Hold retrospective with development team.
